### 2. 风险评估
- 开发难度: Low
- 架构风险: Low
- 关键风险点:
  - 图片体积与分辨率失控会导致 LCP 回退。
  - Lightbox 键盘可达性与焦点管理若实现粗糙，会造成可访问性缺陷。
  - 配置化数据若无字段约束，后续内容维护可能引入空标题/缺失 alt 等质量问题。

### 4. 实施建议
- 资源策略: 统一图片清单（JSON/常量表）+ 构建期校验（URL、alt、尺寸、可选文案字段）。
- 性能策略: 提供 `srcset/sizes`，首屏外全部懒加载；优先 WebP，必要时回退 JPG/PNG。
- 交互策略: Modal 实施焦点陷阱、Esc 关闭、遮罩点击关闭、滚动锁定与恢复。
- 失败降级: 固定占位骨架 + 失败文案，确保网格高度稳定，避免 CLS。
- 验收补充: 将“无明显回退”量化为阈值（如 LCP、CLS、Accessibility 分数基线）。
